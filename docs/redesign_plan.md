# hub.pl3.kr 메인 페이지 디자인 개편 작업계획서

본 계획서는 `hub.pl3.kr` (index.html) 페이지의 구식 UI/UX를 개선하고, 1인 개발자이자 15년차 백엔드 엔지니어로서의 전문성이 돋보이도록 프리미엄 웹 디자인 시스템을 적용하기 위한 작업 계획입니다.

---

## 1. 디자인 컨셉: **Premium Developer Dashboard (Glassmorphic Neo-Noir)**

기존의 단순 텍스트 중심 터미널 스타일을 계승하되, 최신 웹 트렌드인 **글래스모피즘(Glassmorphism)**, **그라디언트 앰비언트 글로우(Ambient Glow)**, **정교한 마이크로 애니메이션**을 결합하여 신뢰감 있고 프로페셔널한 느낌을 줍니다.

### 주요 디자인 방향
- **컬러 팔레트 (Curated Palette)**:
  - **Dark Mode (Default)**: Deep Slate Gray (`#090d12`) 바탕에 매끄러운 Glassmorphic Panel (`rgba(17, 22, 29, 0.7)`)과 Cyber Blue (`#3b82f6`), Emerald Green (`#10b981`) 포인트 컬러 사용.
  - **Light Mode**: 부드러운 오프화이트/실버 그레이 바탕에 깔끔한 보더 라인과 차분한 블루 톤 포인트.
- **타이포그래피 (Typography)**:
  - 헤드라인: 트렌디하고 볼드한 `Outfit` 폰트 적용.
  - 본문: 가독성이 뛰어난 프리미엄 본문용 폰트 `Pretendard` 적용.
  - 코드 및 메타데이터: 고정폭 폰트 `JetBrains Mono` 유지하여 개발자 아이덴티티 보존.
- **인터랙션 (Interactions)**:
  - 마우스 포인터를 따라 움직이거나 부드럽게 일렁이는 배경 앰비언트 그라디언트(Ambient Glow) 효과.
  - 카드 요소 마우스 호버 시 3D 입체감(Floating)과 테두리 그라디언트 하이라이트(Glow Border).
  - 서비스 상세 보기(아코디언) 토글 시 부드럽게 슬라이딩되는 CSS Transition 최적화.

## 2. 화면 구성 및 개선 사항

### ① 상단 네비게이션 바 (Sticky System Bar)
- **현상**: 단순 블랙 배경의 투박한 텍스트 바.
- **개선**:
  - 상단에 고정(Sticky)되면서 스크롤 시 뒷배경이 비치는 반투명 블러(Glassmorphism) 효과 강화.
  - 시스템 상태 표시등(Signal Pulse)의 애니메이션을 더 정교하게 수정하고, 텍스트가 깜빡이는 커서 효과 고도화.
  - 테마 토글 버튼 클릭 시 부드러운 아이콘 회전 효과(해/달 애니메이션) 추가.

### ② 히어로 섹션 (Hero & Stats)
- **현상**: 단순한 텍스트 줄바꿈 및 강조색.
- **개선**:
  - "혼자 설계하고, 혼자 운영합니다." 타이틀 부분에 **그라디언트 텍스트(Gradient Text)** 및 핵심 키워드 순환 타이핑 애니메이션 적용.
  - 1인 개발/운영 통계(Stat Dashboard)를 계기판이나 카드 형태로 묶어 시각적 중요도 확보.

### ③ 게이트웨이 카드 (Gateway Cards)
- **현상**: 투박한 테두리의 3열 그리드 카드.
- **개선**:
  - 카드가 공중에 떠오르는 듯한 그림자 및 부드러운 스케일 업 애니메이션.
  - '준비중(soon)'인 카드는 흐릿하게 블러(Blur) 처리하되, 마우스를 올리면 세련된 메탈릭 그라디언트와 '알림 받기' 등 상호작용 가능한 느낌을 주는 플레이스홀더 UI로 변경.

### ④ 서비스 카드 그리드 (Running Services)
- **현상**: 밋밋한 회색 테두리 카드, 문제/해결책/임팩트가 단순 텍스트로 노출.
- **개선**:
  - 각 서비스의 성격에 맞는 로고 아이콘(혹은 세련된 SVG 플레이스홀더 아이콘) 추가.
  - 기술 스택 배지(Stack Tag)에 브랜드 고유 컬러 부여 (예: React -> Cyan, PHP -> Indigo, PostgreSQL -> Blue 등).
  - 아코디언 열림/닫힘 동작 시 꺾쇠 기호가 회전하며 내부 영역이 미끄러지듯 노출되는 가속도 기반 애니메이션 적용.

## 3. 구현 기술 및 제약 사항
- **Framework**: 기존 시스템과의 호환성 및 무중단 배포를 위해 프레임워크(React, Next.js 등)를 도입하지 않고 **순수 HTML, Vanilla CSS, Vanilla JS** 구조를 유지합니다. (빌드 과정 없이 즉시 반영 가능하도록 최적화)
- **SEO & 성능**:
  - Semantic HTML 마크업 준수 (Section, Article, Header, Footer 분할).
  - CSS 최적화를 통해 렌더링 차단 리소스를 최소화하고, Google Lighthouse 기준 성능 95점 이상 달성 목표.
  - 반응형 웹 디자인(RWD)을 완벽 지원하여 모바일, 태블릿, 데스크톱 전 기기에서 최적의 해상도로 노출.

## 4. 작업 일정 및 검증 계획

### 1단계: 마크업 및 스타일 시스템 구축 (1일)
- 폰트 로드 및 CSS 변수(Theme Variables) 설정.
- 배경 그라디언트 및 글로벌 애니메이션 정의.

### 2단계: 구성 요소 개별 디자인 및 인터랙션 구현 (1일)
- 네비게이션, 히어로 대시보드, 게이트웨이 카드, 서비스 아코디언 카드 스타일 리뉴얼.
- 호버 및 트랜지션 애니메이션 고도화.

### 3단계: 테마 전환 및 상태 업데이트 스크립트 튜닝 (0.5일)
- 다크/라이트 모드 완전 전환 및 로컬 스토리지 연동 체크.
- KST 시계 및 가동 서비스 개수 실시간 연동 스크립트 정리.

### 4단계: 크로스 브라우징 및 반응형 검증 (0.5일)
- Chrome, Safari, Firefox 및 모바일 브라우저 환경 테스트.
- 성능 지표 측정 및 최종 퍼블리싱.
